NFS 서버 구축 및 보안 설정

Linux 기반 NFS 서버와 클라이언트 환경을 구성하고, 접근 제어와 권한 관리로 안전한 파일 공유를 구현한 프로젝트입니다.

프로젝트 개요

  • 프로젝트명: Cyber Security Full Project – NFS 서버 보안 구성 파트
  • 수행기간: 2025.09.01 ~ 2025.09.14
  • 역할: NFS 서버·클라이언트 설정, 접근 제어 정책 수립, 마운트 테스트 및 검증

서비스 간 파일 공유가 필요한 환경에서, 불필요한 권한 노출 없이 안전하게 스토리지를 공유하기 위해 NFS(Network File System)를 이용한 공유 디렉터리를 구성하고 보안 설정을 적용하는 것을 목표로 했습니다.

구현 상세

1. NFS 서버 구성

  • NFS 패키지를 설치하고, 공유할 디렉터리를 생성한 뒤 /etc/exports 파일에 허용 클라이언트와 권한 옵션을 정의했습니다.
  • 특정 IP/서브넷만 접근하도록 제한하고, 읽기/쓰기 권한과 root_squash 등 옵션을 적용해 불필요한 권한 상승을 방지했습니다.

2. 클라이언트 마운트 및 자원 확인

  • 클라이언트 측에서 NFS 유틸리티를 설치하고, 서버에 정의된 공유 디렉터리를 지정 경로에 마운트하여 접근이 가능한지 확인했습니다.
  • 마운트 후 파일 생성·수정·삭제 테스트를 통해 권한이 의도한 대로 동작하는지, 로그에 접근 내역이 남는지 검증했습니다.

보안 설정 및 테스트 결과

  • 허용된 IP가 아닌 다른 호스트에서 접근을 시도했을 때, 마운트가 차단되는지 확인하여 접근 제어가 정상적으로 작동함을 검증했습니다.
  • root_squash 옵션 적용 상태에서 root 계정으로 파일 작업을 시도해도 서버 측에서는 일반 사용자 권한으로 기록되는 것을 확인했습니다.
  • 공유 디렉터리 권한을 과도하게 넓히는 경우, 다른 계정으로도 민감 파일에 접근 가능해지는 위험성을 테스트하며 적절한 권한 설계의 중요성을 체감했습니다.

느낀 점 및 활용 포인트

단순히 NFS를 “공유 디렉터리 도구”로 사용하는 것에서 그치지 않고, IP 기반 접근 제어, 권한 옵션, 로그 확인까지 함께 설계해야 보안상 안전한 파일 공유 환경을 만들 수 있다는 점을 배웠습니다. 이후 서버를 설계할 때 공유 스토리지 사용이 필요한 경우, 초기 설계 단계에서부터 NFS 권한과 접근 정책을 함께 고려하는 습관을 갖게 되었습니다.
