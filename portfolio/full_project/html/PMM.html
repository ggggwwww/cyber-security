PMM을 활용한 DB 성능 및 쿼리 모니터링

Percona Monitoring and Management(PMM)으로 데이터베이스 성능과 쿼리 품질을 시각화하고, 보안 진단과 장애 대응에 활용할 수 있는 모니터링 환경을 구축한 프로젝트입니다.

프로젝트 개요

  • 프로젝트명: Cyber Security Full Project – PMM DB 모니터링 파트
  • 수행기간: 2025.09.01 ~ 2025.09.14
  • 역할: PMM 서버·클라이언트 연결, 대시보드 구성, 부하 테스트 및 분석

보안 위협뿐 아니라 성능 저하와 쿼리 병목 역시 서비스 가용성에 큰 영향을 준다고 판단하여, DB 상태와 쿼리 실행 현황을 실시간으로 확인할 수 있는 PMM 환경을 구축하는 것을 목표로 했습니다.

구현 상세

1. PMM 클라이언트 등록

  • PMM 서버를 구축한 뒤, DB가 동작하는 호스트에 PMM Client를 설치하고 서비스 및 메트릭 수집을 위한 에이전트를 등록했습니다.
  • LogAnalyzer가 사용하는 DB 호스트를 PMM에 연결하여, 로그 분석 시스템과 연동된 DB의 성능 지표를 함께 모니터링할 수 있도록 했습니다.

2. DB·쿼리 분석 대시보드

  • CPU·메모리·디스크 I/O, 연결 수, 쿼리 응답 시간, Slow Query Log 등을 한눈에 볼 수 있는 기본 대시보드를 활용했습니다.
  • 의도적으로 부하 테스트 스크립트를 실행해 쿼리 지연이 발생하는 구간을 찾고, 인덱스 추가나 쿼리 튜닝이 필요한 부분을 식별했습니다.

테스트 시나리오 및 결과

  • 동시에 많은 요청을 보내는 부하 테스트를 수행해, 특정 시점에 CPU 사용률과 쿼리 응답 시간이 급증하는 패턴을 확인했습니다.
  • Slow Query 상위 목록을 분석해, 불필요하게 전체 테이블 스캔을 유발하는 쿼리를 찾아 인덱스를 추가하거나 조건을 수정하는 개선안을 도출했습니다.
  • 성능 이슈가 발생했을 때 PMM 지표와 시스템 로그를 함께 보는 방식으로, 장애 원인을 더 빠르게 추적할 수 있음을 검증했습니다.

느낀 점 및 활용 포인트

보안 관점에서의 취약점만이 아니라, DB 성능과 쿼리 품질 역시 서비스 가용성을 위협하는 요소라는 점을 PMM을 통해 체감했습니다. 이후에는 모의해킹이나 취약점 진단을 수행할 때도, 공격 전후의 성능 지표와 로그 변화를 함께 확인하여 “보안 + 성능” 관점에서 시스템을 바라보는 습관을 갖게 되었습니다.
