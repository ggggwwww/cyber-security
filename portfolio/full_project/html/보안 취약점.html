1. 비밀번호 설정

"/etc/security/pwquality.conf"||"/etc/login.defs" 파일에서 Linux 시스템의 비밀번호 정책에 관한 설정을 수정할 수 있습니다.

해당 화면에서 설정한 비밀번호 복잡도 값은 다음과 같습니다.

  • 1. minlen(비밀번호의 최소 길이): 10
  • 2. dcredit(비밀번호의 최소 숫자 포함 개수): 2
  • 3. ucredit(비밀번호의 최소 영대문자 포함 개수): 0
  • 4. lcredit(비밀번호의 최소 영소문자 포함 개수): 0
  • 5. ocredit(비밀번호의 최소 특수문자 포함 개수): -2

해당 화면에서 설정한 비밀번호 정책

  1. PASS_MAX_DAYS(비밀번호 만료 기간): 90일 후 만료

다음과 같이 비밀번호 정책 설정을 완료하였습니다, 이를 통해 시스템의 보안성을 강화할 수 있습니다.

2. BoF(Buffer Over Flow) 발생시키기

BoF(Buffer Over Flow)는 버퍼의 크기를 초과하는 데이터를 입력하여 인접한 메모리 공간을 덮어쓰는 취약점입니다. 이를 통해 악의적인 사용자가 시스템에 대한 권한을 획득하거나, 시스템을 불안정하게 만들 수 있습니다.

위의 예시에서는 8바이트 크기의 버퍼(char name)에 12바이트 이상의 데이터를 입력하여 BoF를 발생시켰습니다. 이를 통해 노출되어서는 안되는 char secret 변수의 값이 덮어써지게 되었고, 이를 통해 중요한 정보를 탈취할 수 있었습니다.

3. setuid-bit 조작을 통한 권한 상승 시도

setuid는 파일의 소유자가 아닌 사용자가 해당 파일을 실행할 때, 파일 소유자의 권한으로 실행되도록 하는 특수한 권한 비트입니다. 이를 통해 일반 사용자가 시스템 관리자(root) 권한이 필요한 작업을 수행할 수 있습니다.

위의 예시에서는 "chmod 4777 test.txt" 명령어를 통해 test.txt 파일에 setuid 비트를 설정하였습니다. 이를 통해 일반 사용자가 test.txt 파일을 실행할 때, 파일 소유자의 권한으로 실행되도록 하였습니다. 이를 통해 일반 사용자가 root 권한으로 시스템에 접근할 수 있게 되었습니다.

4. btmp&wtmp 로그 분석을 통한 시스템 취약점 확인

btmp와 wtmp는 Linux 시스템에서 로그인 및 로그아웃 기록을 저장하는 파일입니다. btmp는 실패한 로그인 시도를 기록하며, wtmp는 성공한 로그인 시도를 기록합니다. 이를 통해 시스템 관리자는 사용자 활동을 모니터링하고, 보안 위협을 감지할 수 있습니다.

5. Sticky Bit 설정

Sticky Bit는 디렉토리에 설정되는 특수한 권한 비트로, 해당 디렉토리 내의 파일을 삭제하거나 이름을 변경할 수 있는 사용자를 제한합니다. 일반적으로 /tmp 디렉토리에 설정되어 있으며, 이를 통해 여러 사용자가 공유하는 디렉토리에서 파일의 무단 삭제를 방지할 수 있습니다.

위의 예시에서는 test.txt 파일에 Sticky Bit를 설정하였습니다. 이를 통해 test.txt 파일이 위치한 디렉토리 내에서 해당 파일을 삭제하거나 이름을 변경할 수 있는 사용자를 제한하였습니다. 이를 통해 시스템의 보안성을 강화할 수 있습니다.
