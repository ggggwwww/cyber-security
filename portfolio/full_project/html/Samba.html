Samba 파일 공유 서버 구축 및 보안 설정

Linux Samba 서버를 이용해 윈도우/리눅스 간 파일 공유 환경을 구축하고, 접근 제어와 권한 관리를 통해 내부 정보 유출 위험을 줄이는 구성을 실습한 프로젝트입니다.

프로젝트 개요

  • 프로젝트명: Cyber Security Full Project – Samba 파일 공유 보안 파트
  • 수행기간: 2025.09.01 ~ 2025.09.14
  • 역할: Samba 서버 설정, 공유 폴더 권한 설계, 클라이언트 접근 테스트

조직 내에서 파일을 공유할 때 편의성과 보안성을 동시에 확보하기 위해, Samba를 기반으로 한 파일 서버를 구축하고 계정·권한·로그 설정을 적용하는 것을 목표로 했습니다.

구현 상세

1. Samba 서버 설정

  • Samba 패키지를 설치한 뒤 smb.conf 파일에서 공유 이름, 경로, 접근 가능한 사용자/그룹, 읽기/쓰기 권한을 정의했습니다.
  • 익명 접근을 차단하고, 특정 사용자에게만 쓰기 권한을 부여하도록 설정해 불필요한 파일 변조를 방지했습니다.
  • 로그 레벨을 조정해 접속·파일 작업 이력을 기록하고, 보안 사고 발생 시 추적 가능하도록 구성했습니다.

2. 서버 측 파일 생성 및 권한 확인

  • 서버 측 공유 디렉터리에서 테스트 파일을 생성하고, 소유자와 권한이 의도한 설정대로 적용되는지 확인했습니다.
  • 서버 로컬 계정과 Samba 계정을 분리 관리하며, 권한 매핑이 올바르게 이뤄지는지 검증했습니다.

3. 클라이언트 접속 및 파일 공유 테스트

  • Windows/리눅스 클라이언트에서 Samba 공유 폴더에 접속하여, 계정·비밀번호 인증 후 파일 열기·저장 테스트를 진행했습니다.
  • 읽기 전용 계정과 쓰기 가능한 계정을 나누어, 각 계정이 실제로 허용된 작업만 수행할 수 있는지 확인했습니다.

보안 테스트 및 결과

  • 허용되지 않은 사용자/호스트에서 공유 폴더에 접속을 시도했을 때, 인증 실패 및 접근 거부가 정상적으로 로그에 기록되는지 확인했습니다.
  • 공유 디렉터리의 권한을 과하게 개방했을 때, 다른 사용자 계정으로 민감 파일에 접근 가능한 위험성을 테스트하며 최소 권한 설정의 중요성을 실감했습니다.

느낀 점 및 활용 포인트

Samba와 같은 파일 공유 서비스는 편리하지만, 계정·권한·로그 설정이 부족하면 내부자에 의한 정보 유출 통로가 될 수 있다는 점을 직접 확인했습니다. 이후 서버나 사내 인프라를 설계할 때, 파일 공유가 필요한 경우에는 항상 접근 제어 리스트와 감사 로그를 함께 설계하는 것을 기본 원칙으로 삼게 되었습니다.
