Zabbix를 활용한 서버·서비스 모니터링

Zabbix 서버와 에이전트를 구성해 서버 자원과 서비스 상태를 상시 모니터링하고, 임계치 기반 알람으로 장애를 사전에 감지하는 환경을 구축한 프로젝트입니다.

프로젝트 개요

  • 프로젝트명: Cyber Security Full Project – Zabbix 모니터링 파트
  • 수행기간: 2025.09.01 ~ 2025.09.14
  • 역할: Zabbix 서버 구축, 호스트 등록, 트리거/알람 설정, 포트 포워딩 테스트

보안 이벤트뿐 아니라 서버 성능과 서비스 가용성을 함께 관리하기 위해 Zabbix를 활용한 통합 모니터링 환경을 구축하고, 임계치 알람을 설정하는 것을 목표로 했습니다.

Zabbix 서버 및 호스트 설정

1. Zabbix 서버 배포

  • Zabbix 서버와 웹 인터페이스를 설치하고, DB 연동 및 기본 템플릿을 적용해 모니터링 환경의 기초를 구성했습니다.
  • 관리 콘솔에서 시간대·언어·알림 설정을 초기화하여 이후 호스트 등록과 트리거 설정이 가능하도록 준비했습니다.

2. 호스트 생성 및 에이전트 연결

  • 모니터링 대상 서버에 Zabbix Agent를 설치하고, 서버 IP와 호스트명을 설정한 뒤 Zabbix 콘솔에서 호스트를 등록했습니다.
  • 템플릿을 적용해 CPU, 메모리, 디스크, 프로세스, 네트워크 트래픽 등 기본 지표를 자동 수집하도록 구성했습니다.

포트 포워딩 및 에이전트 분석 화면

1. 포트 포워딩 설정

  • 방화벽 환경에서 Zabbix Agent의 기본 포트(10050/TCP)가 서버와 통신할 수 있도록 포트 포워딩 규칙을 설정했습니다.
  • 포워딩 전·후 패킷 캡처와 연결 테스트를 통해 통신 상태를 확인하고, 모니터링 데이터가 정상적으로 수집되는지 점검했습니다.

2. 에이전트 지표 분석

  • CPU 사용률, 메모리 점유, 디스크 사용량, 프로세스 수 등 주요 지표를 그래프로 시각화하여, 부하 패턴을 확인했습니다.
  • 의도적으로 부하를 발생시키는 스크립트를 실행하고, 지표 변화와 트리거 알람 발생 여부를 함께 검증했습니다.

테스트 결과 및 인사이트

  • CPU·메모리 사용률에 임계치를 설정하여, 일정 수준 이상 사용이 지속될 때 알람이 생성되는지 확인했습니다.
  • 서비스 장애 상황(예: 특정 데몬 중지)을 의도적으로 발생시켜, Zabbix가 다운된 서비스를 감지하고 이벤트를 기록하는 과정을 검증했습니다.
  • 이 실습을 통해 성능·가용성 모니터링 정보를 보안 이벤트와 함께 보는 것이, 실제 사고 분석과 재발 방지에 중요하다는 점을 체감했습니다.

느낀 점 및 활용 포인트

Zabbix를 구축하고 직접 지표·트리거를 설정해 보면서, 서버나 서비스의 이상 징후를 사람이 느끼기 전에 시스템이 먼저 알려줄 수 있도록 하는 것이 중요하다는 점을 배웠습니다. 이후 보안 인프라를 설계할 때도, 침입 탐지뿐 아니라 성능·가용성 모니터링까지 함께 고려해 장애와 보안 이슈를 종합적으로 관리하는 것이 필요하다는 점을 기준으로 삼게 되었습니다.
